Workstation Arc/Info Interchange Import Error

I have been working on updating some data that was published using Arc/Info Workstation Export (.e00) files, converting them to shapefiles, or in the case of annotation, to a geodatabase feature class. While converting some INFO tables, I have received this error:Unable to encode the 38th item in the 65th record for the INFO fileIt turns out that Workstation can create export files that contain values that did not fit within their field–I’m guessing that somehow you are actually able to enter invalid values into those fields but I did not confirm that.
